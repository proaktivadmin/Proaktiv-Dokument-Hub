# Directory Structure Map

## Root Folders
-   **`client/`**: The React Frontend application (Vite).
    -   `src/components/`: Reusable UI components (GlassInput, etc.).
    -   `src/MigrationView.jsx`: The specific UI for the Migration Factory.
-   **`documentation/`**: Project documentation.
    -   `archive/`: Historic verification logs and plans (`v1_...`).
    -   `prompts/`: System prompts for AI agents.
-   **`library/`**: **CRITICAL** - The persistent data store.

## The Library (`library/`)
This folder acts as the "database" for the application.

### 1. `System/`
Contains the "Master Templates" and CSS definition files.
-   `styles/`:
    -   `vitec-structure.css`: Layout and resets.
    -   `theme-base.css`: Default variables.
    -   `theme-proaktiv.css`: **Primary Brand Theme.**
-   `Vitec Stilark.html`: The HTML wrapper used for styling previews.

### 2. `Legacy_Import/`
**Input Folder:** Drop old Vitec templates here that need to be modernized.
-   *Action:* Use "Migration Factory" in the settings menu to process these files.

### 3. `Ready_For_Export/`
**Output Folder:** Contains the sanitized, modernized templates generated by the Migration Factory.
-   *Action:* Copy these files back into Vitec Next.

### 4. `Akseptbrev`, `Oppdragsavtale`, etc.
Organizational folders for the actual document templates. New categories created in the UI appear here as folders.
